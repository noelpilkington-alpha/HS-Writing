## Amendment 1 — Memorize-It + a carried mnemonic to automaticity (the #1 gap)

**The gap (confirmed from four independent angles).** C1 gates on a **cold, timed, no-reference** task (1.5 bridge, 4.6 EI gate, 8.5 EII gate) but only fades the *visible* scaffold — it never installs the *internal* one. SRSD's own documented failure case is the student with nothing to retrieve at the test. C1 names STOP+DARE/TREE but **no lesson installs a strategy to retrieval criterion.** (Verify_Architecture_LearningSci §"missing #1".)

**This is an INTEGRATION, not a new design.** The mechanism already exists in the SRSD proposal docs — it just isn't wired into C1:

- **Carried mnemonic for the expository-primary mode = FACT** (`SRSD Activity Specs` Activity 6; `SRSD Integration Brainlift` Truth 4, Node S4). FACT at essay scale: **F**ocus (controlling idea / intro) · **A**dd details (key facts from the passage, → body paras) · **C**larify (explain each detail — *this is the evidence-explanation lever, the C5 3-vs-2 move, C1's "reason not restate"*) · **T**ie up (conclusion). Use FACT exactly as the house defines it — "acronyms are vehicles, stay consistent" (Integration Brainlift Insight 9).
- **STOP+DARE** = the argument upgrade, adopted at **Unit 3** (DARE's **R** = reject-the-other-side = the counterargument move C1 3.3 already teaches). Same refresh model.
- **NOT** POW+TREE (generic SRSD research vocabulary, not the AlphaWrite house system) and **NOT** R.A.P.S. (explicitly *removed* from the SRSD specs, line 476–483, as a reading-comprehension skill redundant with FACT+PROVE).

**The activity template C3 instantiates** (`SRSD Activity Specs` Activity 5/6 — already specified): (1) prompt-type discrimination → (2) passage read + comprehension check → (3) mnemonic-field planning → (4) graded draft, plan visible → (5) mnemonic self-check → (6) feedback/revise.

**The Memorize-It fade** (`SRSD Activity Specs` "Scaffold fade sequence," 4 phases): full fields+descriptions → **labels only** → **planning optional** → **mastery gate (no fields, one shot, test conditions)**. This *is* the Heavy→Medium→Light→None fade C1 already names; Amendment 1 binds it to the FACT mnemonic so the internal scaffold is what fades in, and reaches **internal (no cue card) before the timed gates (U7/U8)**.

**HS-specific deltas (the one real seam — C1 is G9-10, the SRSD specs are G3-8):**
- **Deliver FACT as "recall-or-learn" refresh, not slow first-teach.** Robust whether or not the SRSD G6-8 proposal ships (it is *only a proposal*, and stops at G8 — no G9-12 placement exists). A student arriving with FACT gets a fast refresh; one without it gets a real (but brisk) teach. (Decision 2026-06-18.)
- **C1's comprehension pre-read is scored SEPARATELY** (C5/SPOV 5) — a sharper version of the Activity-Specs "comprehension check (MC)" step. Keep C1's separate scoring; it lets the grader tell "didn't read" from "can't write."

**Open question (logged, not solved now):** full **G3–12 vertical articulation** — what HS actually inherits once SRSD ships in G6-8 (redundancy vs. gap at the seam). Revisit when the SRSD G6-8 proposal is built and a real student has been through both bands.

**C1 edits this implies:** add an explicit "learn/refresh + memorize FACT to criterion" beat in U1 (around 1.1–1.4); tag each unit's core production lesson with its mnemonic + fade phase; state that the cue card is internal before the U7/U8 timed gates.

## Amendment 2 — Cross-unit retrieval / interleaving + a light fluency thread

**The gap.** C1 fades scaffolds *within* each thread but never **re-tests a prior essay type** until U8. So the EII gate (8.5) is the *first* time a student faces cumulative + interleaved + timed + cold all at once — anti-cognitive-load, and a transfer risk. (Verify_Architecture_LearningSci §"missing #2".)

**Fix (small, additive):**
- **Interleave a prior essay type into later units' practice** — e.g., a brief expository (E1) rep surfaces inside U3/U5, an argument rep inside U5/U7 — so the gate isn't the first cumulative experience. Spacing + interleaving are well-evidenced (Kirschner corpus); this is retrieval practice across units, not new content.
- **Light sentence-level fluency thread.** C1's sentence-combining (4.2) is a one-off SCR (confirmed in C4). Thread a brief sentence-combining/expansion beat into the Polish stage of essay reps toward timed throughput (TWR sentence-combining d≈0.50; supports the U7/U8 timed demand).

**C1 edits this implies:** add a one-line "cumulative retrieval" note to U3/U5/U7 practice reps; add a recurring Polish-stage fluency beat (parallels the EII diction/tone Polish beat C2 already added).

## Amendment 3 — Working claim at 1.1 + reword "reasoning before thesis"

**The issue.** C1 §31–33 states "reasoning is taught before the thesis." Verified literally false and contradicted by every external curriculum *and the brainlift's own structural tables*: a warrant/reasoning move is **by definition downstream of a claim** (Toulmin: the warrant "links the grounds to the claim") — you cannot write "this shows [the claim]" before a claim exists. (Verify_ReasoningBeforeThesis: SUPPORTED-WITH-CAVEAT.)

**Fix (precise, small):** the defensible and correct version is **working claim → reasoning → refined controlling idea**:
- Allow a **provisional/working claim at 1.1** (C1's own 1.1 mechanics already assume one — "given a claim + evidence, write the reasoning").
- Reserve **refinement** of the controlling idea for **1.4** (already there).
- Reword the rationale: *"developing the thesis is the problem, not stating it — so we teach the reasoning that earns a controlling idea before we teach refining the controlling idea itself"* — NOT "reasoning before any claim."
- Do **not** claim external curricula sequence reasoning ahead of the position; they uniformly do the opposite.

**C1 edits this implies:** reword the U1 design-rationale line; add "(working claim)" to 1.1's move description.
