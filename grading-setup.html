Grading Engine Setup

Enable AI grading so interactive lessons can score student writing.

Option A: Use Hosted Grading (No Setup Required)

The grading engine is already running as a hosted service. To connect your lessons to it, open your browser's developer console and paste these two lines:

How to Open the Console

  • Chrome / Edge: Press F12 (or Ctrl+Shift+J on Windows, Cmd+Option+J on Mac), then click the Console tab.
  • Firefox: Press F12 (or Ctrl+Shift+K), then click the Console tab.
  • Safari: Enable the Develop menu in Preferences > Advanced, then press Cmd+Option+C.

Paste These Two Lines

localStorage.setItem('gradingApiUrl', 'https://hs-writing-grading.onrender.com');
localStorage.setItem('gradingApiKey', 'IFyOA6KGkuZi6eQ7Ot952nfP9XMhjUuPb37HUT24YiM=');

Press Enter. That's it — you only need to do this once per browser. The setting persists across sessions. All lessons will now grade student writing through the hosted server.

Note: The hosted server runs on Render's free tier. The first grading request after 15 minutes of inactivity may take 30–60 seconds while the server wakes up. After that, responses are 5–15 seconds.

Option B: Run Locally (Advanced)

If you prefer to run the grading engine on your own machine (for example, to use your own API key or work offline), follow the steps below.

1. What You Need

  • Python 3.10+ — download from python.org if you don't have it. Verify with python --version.
  • An Anthropic API key — sign up at console.anthropic.com and create an API key. The grading engine uses Claude to evaluate student writing.
  • pip — comes with Python. Verify with pip --version.

2. Download

The grading engine lives in the api/ folder of the course repository. You need all the files in that folder.

Option A: Clone the repo (recommended)

git clone https://github.com/noelpilkington-alpha/HS-Writing.git

This downloads the entire repository. The grading engine files are in the api/ subdirectory.

Option B: Download as ZIP

Go to the GitHub repository page, click the green Code button, then Download ZIP. Extract the ZIP and locate the api/ folder inside.

3. Setup Steps

1

Clone or download the repository (see above).

2

Open a terminal and navigate to the api folder:

cd HS-Writing/api
3

Install dependencies:

pip install -r requirements.txt

This installs FastAPI, Uvicorn, the Anthropic SDK, python-dotenv, and Pydantic.

4

Create a .env file inside the api/ folder with your API key:

ANTHROPIC_API_KEY=your-key-here
Important: Replace your-key-here with your actual Anthropic API key. Never commit this file to version control.
5

Start the server:

python -m uvicorn main:app --host 127.0.0.1 --port 8003

You should see output like Uvicorn running on http://127.0.0.1:8003. Keep this terminal window open.

6

Open lessons in your browser. The interactive HTML lessons will automatically connect to the local grading API at http://127.0.0.1:8003. Student writing submissions will be scored in real time.

4. Files Included

The api/ folder contains the following files:

FileDescription
main.py FastAPI application entry point. Defines API endpoints for grading student writing across all 4 courses (A1, A2, B1L, B2).
models.py Pydantic data models for all 6 scoring models: Criteria Checklist (A), AP Rubric (B), Hybrid (C), Discrimination (D), Planning (E), and Revision (F).
consensus.py 3-run consensus engine. Runs each grading call 3 times and resolves disagreements via unanimous, majority, or judge-call logic.
grading_rubrics.py Per-lesson, per-task grading rubrics for all courses. Defines criteria, common pitfalls, word-count minimums, and max scores.
course_rubrics.py Course-specific rubrics for A2, B1L, and B2. Merged into the main rubric registry at import time.
grading_prompts.py Prompt templates sent to Claude for grading. Implements feedback principles: specific, no cheerleading, wise feedback, one concrete next step.
ap_scorer.py AP Rubric scorer (Model B). Scores essays on the official AP English Language rubric: Thesis (0–1), Evidence & Commentary (0–4), Sophistication (0–1).
revision_scorer.py Revision scorer (Model F). Compares original and revised text to evaluate whether changes are substantive or surface-level.
passage_bank.py Stores source passages by ID (SP-001, SYN-07, etc.) for grading prompts that need the original text as context.
course_review_agent.py Standalone review tool. Reads lesson HTML, sends it to Claude for pedagogical evaluation, and generates a structured report.
requirements.txt Python dependencies: FastAPI, Uvicorn, Anthropic SDK, python-dotenv, Pydantic.

5. Easy Setup with Claude Code

If you're not comfortable with the command line, Claude Code can do all of the setup for you. Claude Code is an AI assistant that runs commands on your computer. It will install dependencies, create files, and start the server — you just approve each step.

What is Claude Code?

Claude Code is a tool made by Anthropic (the same company behind the grading engine). You type what you want in plain English, and it runs the necessary terminal commands for you. You can review and approve every action before it happens.

One-Time Setup

1

Install Node.js (required to run Claude Code).

Download and install from nodejs.org. Choose the LTS (Long Term Support) version. Run the installer and accept the defaults.

2

Install Claude Code. Open a terminal (on Windows: search for "Terminal" or "Command Prompt" in the Start menu; on Mac: open "Terminal" from Applications > Utilities) and paste this command:

npm install -g @anthropic-ai/claude-code
3

Get an Anthropic API key. Sign up at console.anthropic.com and create an API key. You will need this key for both Claude Code and the grading engine.

Running the Grading Server

Once Claude Code is installed, open a terminal, navigate to the repository folder, and type:

claude

This starts Claude Code. Then paste the following message:

Start the grading API server for me. The server code is in the api/ folder. Install the Python dependencies from requirements.txt if they aren't already installed. Create a .env file in the api/ folder with my Anthropic API key if one doesn't exist (ask me for the key). Then start the server with: python -m uvicorn main:app --host 127.0.0.1 --port 8003

Claude Code will walk you through each step. It will ask for your approval before running each command. Just type y (yes) to approve, or n (no) to skip a step.

What to Expect

  • Claude Code will check if Python is installed and help you install it if not.
  • It will install the required Python packages (FastAPI, Uvicorn, etc.).
  • It will create the .env file with your API key.
  • It will start the grading server. You should see: Uvicorn running on http://127.0.0.1:8003.
  • Keep the terminal window open while you use the lessons.

Next Time

After the first setup, you only need to start the server. Open a terminal, navigate to the repository, run claude, and type:

Start the grading server on port 8003.

6. Troubleshooting

Port 8003 already in use

Another process is using the port. Either stop that process or run on a different port:

python -m uvicorn main:app --host 127.0.0.1 --port 8004

If you use a different port, open your browser console and run:

localStorage.setItem('gradingApiUrl', 'http://localhost:8004')

This overrides the default for all lessons without editing any files. The setting persists across browser sessions.

ANTHROPIC_API_KEY not set / 401 Unauthorized

Make sure the .env file exists inside the api/ folder and contains a valid key. Double-check there are no extra spaces or quotes around the key value.

ModuleNotFoundError

Dependencies are not installed. Run pip install -r requirements.txt again. If you have multiple Python versions, try pip3 or python -m pip install -r requirements.txt.

Lessons don't connect to the grading API

Make sure the server is running and accessible at http://127.0.0.1:8003. If you opened the HTML files directly from disk (via file://), some browsers block local API requests. Try serving the lesson files through a local HTTP server:

python -m http.server 8080

Then open http://127.0.0.1:8080 in your browser.

Slow responses

The consensus engine makes 3 parallel Claude API calls per grading request (and sometimes a 4th judge call). Response times depend on Claude API latency. Typical turnaround is 5–15 seconds per submission.

Python not recognized / command not found

Python may not be in your system PATH. On Windows, re-run the Python installer and check "Add Python to PATH." On macOS/Linux, try python3 instead of python.
